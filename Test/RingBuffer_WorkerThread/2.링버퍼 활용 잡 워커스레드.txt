RingBuffer 과제 1

	Enqueue 스레드 x 1
	Dequeue 스레드 x 1

	인 경우 인큐와 디큐는 스레드 동기화 문제가 없어야 합니다.
	이에대한 테스트를 통해서 오류가 없도록 RingBuffer 를 완성 하여 둡니다.






- 메시지 워커 스레드 (액터패턴) 설계 과제

1. RingBuffer - Lock, Unlock 추가.   (멀티스레드를 위해서 )

2. RingBuffer 를 사용하여 잡 메시지를 생성 스레드, 메시지 처리 스레드 테스트 구현



# 메시지 구조 예시.

struct st_MSG_HEAD
{
	short shType;
	short shPayloadLen;
};


// 워커스레드에 일감을 요청하는 방식은  네트워크 메시지와 비슷함
// 그래서 헤더에 일의 종류와, 이에대한 가변적인 데이터를 전달하는 형태가 일반적임

// Type 별로 가상의 일감을 정의.
// Type 에 따라서 추가적인 필요 데이터는 Head 이후에 Len 만큼 삽입.  (문자열 등...)


- 메시지 타입 예시

#define dfJOB_ADD	0
#define dfJOB_DEL	1
#define dfJOB_SORT	2
#define dfJOB_FIND	3
#define dfJOB_PRINT	4	<< 출력 or 저장 / 읽기만 하는 느린 행동
#define dfJOB_QUIT	5	




//-----------------------------------------------
// 컨텐츠 부, 문자열 리스트
//-----------------------------------------------
list<wstring>		g_List;

//-----------------------------------------------
// 스레드 메시지 큐 (사이즈 넉넉하게 크게 4~5만 바이트)
//-----------------------------------------------
CRingBuffer 		g_msgQ;



@ 메인스레드

- 워커스레드 3개 이상 생성.


{  (반복)

	1. 메시지 생성

	메시지 타입을 랜덤하게 생성.
	메시지에 따른 추가적인 데이터 (문자열) 도 랜덤하게 삽입.
	
	{ 메시지 인큐

		데이터를 msgQ 에 인큐.
		이벤트를 통해 워커 스레드 깨우기.
	}

	2. 특정 키가 눌렸다면, 종료 메시지 dfJOB_QUIT 인큐, 워커 스레드 깨우기 & 잡 생성 중지.
	

}


% 메인 스레드는 모든 워커스레드가 끝났다면 종료한다
% 메인 스레드의 루프는 50ms 대기시간 으로 개발
% 50ms 대기시간에 이상이 없으면 점점 줄여 나가는 것으로 테스트.

	

@ 워커스레드 (3개 이상)

- 메인스레드 에서 메시지를 넣지 않았다면 쉬고(블락) 있어야 함.

스레드 시작로그 - ID 출력

{ (반복)

	1. 메시지 큐에서 메시지 뽑음.

	2. 헤더 타입에 맞게끔 

		ADD - 메시지로 온 문자열을 g_List 에 추가.
		DEL - g_List 에서 노드 하나 삭제
		SORT - g_List 를 정렬
		FIND - 임의의 검색
		QUIT - 스레드 종료.	
	
}
	

@ 특정 스레드는 1초마다 현 상태 모니터링 

* 메시지 큐 의 사이즈 (사용량)  출력
* 초당 메시지 전체 처리 수 (TPS) + 스레드 별 처리수
* 초당 타입별 메시지 처리 수 (메시지 별 TPS)

+ 기능

메시지 큐의 사용량이 일정량 이상 도달시 메시지의 생성부를 의도적으로 느리게 유도하는 기능
